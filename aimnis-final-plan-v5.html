위젯 구조 — 제품의 핵심 자산
확정 원칙
위젯 = AIMNIS의 핵심 자산. AIM GUARD 구매 = AIM GUARD 전용 위젯 패키지를 받는 것. 신규 기능 개발 = 새 위젯으로 패키징하여 마켓플레이스에 등록.
공용 위젯
모든 고객 무료 사용. 솔루션 관계없이 공유.
  • 라인차트, 바차트, 도넛차트
  • KPI 카드, 게이지, 테이블
  • 알람 리스트, 상태 카드
모든 솔루션무료
솔루션 위젯
해당 솔루션 구독자만 사용. 다른 솔루션 화면에도 올릴 수 있음.
  • GUARD: 배터리감지, CCTV맵, 작업자안전
  • ECO: 탄소배출, 스마트미터, ESG리포트
  • 구독 늘릴수록 위젯 선택지 증가
구독자 전용교차 사용 가능
프리미엄 위젯
추후 별도 판매. 고급 분석 및 AI 예측 기능.
  • AI 예측 차트
  • 3D 맵 시각화
  • 고급 분석 대시보드
별도 구매추후 출시
03
UX 철학 — 플랫폼 전체 적용
"AI가 묻고, 사용자는 답한다"
개발을 모르는 실무 담당자가 처음부터 끝까지 혼자 완성. 온보딩부터 에디터까지 동일하게 적용.
온보딩 단계
  • AI가 먼저 질문: "상세 기획서가 있으시면 첨부해주세요"
  • AI가 먼저 질문: "브랜드 로고 파일이 있으신가요?"
  • AI가 먼저 질문: "API 명세서가 있으신가요? 없어도 괜찮습니다"
  • 없어도 OK — AI가 최선의 결과 자동 생성
  • 5대 문서를 항목 업로드가 아닌 자연어 대화로 수집
에디터 단계
  • AI가 먼저 개선점 제안 (설정에 따라)
  • 사용자는 승인/거절만으로 작업 완료
  • MVP: 자연어로만 수정 (드래그는 v1.1 이후)
  • AI 개입 수준 3단계 세팅 가능
  • 페이지 추가도 자연어로: "알람 페이지 추가해줘"
수동
내가 물어볼 때만 AI가 답함. IT 담당자용.
반능동 (기본값)
작업할 때 옆에서 제안. 실무 담당자용.
능동
AI가 알아서 분석·개선. 경영진용.
